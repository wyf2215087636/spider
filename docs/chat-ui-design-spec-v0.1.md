# AI研发协作平台 - Chat UI设计规范 v0.1

- 版本：v0.1
- 日期：2026-03-01
- 适用阶段：MVP到M2
- 决策状态：执行基线

## 1. 设计目标

1. 以“聊天驱动协作”为入口，但核心是“项目交付工作台”。
2. 同时承载对话、任务、文档、风险，不做纯聊天产品形态。
3. 提供可追溯证据链（来源、操作、审批）并保持低学习成本。

## 2. 页面信息架构

### 2.1 一级导航
1. Chat协作
2. 项目看板
3. 文档中心
4. 风险与里程碑
5. 系统设置

### 2.2 Chat页面布局（桌面端）
采用三栏布局：
1. 左栏（280px）：Workspace切换、会话列表、过滤器
2. 中栏（自适应）：消息流、卡片化结果、输入区
3. 右栏（360px）：上下文面板（任务、风险、文档、审批）

## 3. 核心交互模型

1. 输入问题 -> Agent响应（流式）-> 结构化结果卡片（可执行动作）
2. 结构化卡片支持“一键落地”：
- 创建任务
- 生成PRD/API/测试文档
- 提交审批
- 加入里程碑
3. 所有输出显示“证据来源”，支持跳转到代码/任务/文档

## 4. 核心组件清单

1. `WorkspaceSwitcher`
2. `ConversationList`
3. `MessageStream`
4. `PromptComposer`
5. `SourceCitation`
6. `ActionCard`（任务拆解、风险摘要、文档草稿）
7. `ApprovalPanel`
8. `ContextDrawer`（当前需求、关联任务、阻塞项）
9. `CommandAuditTimeline`

## 5. 页面状态设计

1. 首次进入：引导创建Workspace和首个会话
2. 空会话：显示快捷意图模板（拆解任务、估时、生成文档）
3. 处理中：流式输出 + 可取消 + 阶段进度（检索中/生成中/校验中）
4. 失败态：错误原因 + 重试 + 降级方案（仅文本输出）
5. 受限态：权限不足时显示审批入口，不隐藏功能

## 6. 视觉风格（非ChatGPT照搬）

### 6.1 设计方向
1. 风格关键词：`专业、冷静、工程感、可审计`
2. 避免通用聊天产品同质化，突出“项目交付”属性

### 6.2 字体
1. 中文：`Noto Sans SC`
2. 英文与数字：`IBM Plex Sans`
3. 代码与日志：`JetBrains Mono`

### 6.3 色彩令牌（建议）
1. `--bg`: `#F6F8FB`
2. `--surface`: `#FFFFFF`
3. `--text-primary`: `#0F172A`
4. `--text-secondary`: `#475569`
5. `--brand`: `#0F766E`（青绿，避免紫色同质化）
6. `--warning`: `#B45309`
7. `--danger`: `#B91C1C`
8. `--border`: `#DCE3EE`

### 6.4 背景与层次
1. 主背景使用浅灰蓝渐变，避免纯平面白底
2. 关键卡片使用轻阴影和边框提升信息分层

### 6.5 动效
1. 首屏加载：分栏渐进显示（200-350ms）
2. 消息流：按段落渐显，不使用花哨粒子动画
3. 卡片动作：执行成功后状态变更动画（120-180ms）

## 7. 响应式策略

1. Desktop（>=1200）：三栏全开
2. Tablet（768-1199）：右栏折叠为抽屉
3. Mobile（<768）：单栏消息流，底部Tab切换“会话/上下文/操作”

## 8. 可用性与可访问性

1. 键盘优先：`Ctrl/Cmd + K` 全局命令、`Enter` 发送、`Shift+Enter` 换行
2. 颜色对比满足 WCAG AA
3. 对重要动作提供二次确认（审批、外部系统写入）
4. 长文本自动分段与摘要，避免信息淹没

## 9. 国际化（中英）要求

1. UI必须支持 `zh-CN` 与 `en-US` 切换
2. 状态文案统一走语言资源，不允许硬编码
3. Chat输入区支持“输出语言”快速切换（中文/English）
4. 任务卡、风险卡、审批卡状态需中英一致映射
5. 页面级日期时间按语言与时区格式化展示

## 10. 前端实现建议

1. 路由：
- `/chat`
- `/chat/:conversationId`
- `/board`
- `/docs`
- `/risks`
- `/settings`
2. 状态管理：
- 服务器状态：`TanStack Query`
- 界面状态：`Zustand` 或 React Context
3. 实时能力：
- 优先 `SSE` 做流式输出
- WebSocket用于后续多用户协作态

## 11. MVP验收标准（UI）

1. 能完成“提问 -> 结构化卡片 -> 一键落地动作”主链路
2. 所有AI输出可见证据引用
3. 移动端可完成核心问答与任务创建
4. 页面首屏可交互时间 <= 3秒（试点环境）
5. 关键操作有审计入口可追溯
6. 语言切换后状态与系统提示全部同步切换（zh-CN/en-US）
